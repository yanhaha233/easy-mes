# 生产级 MVP 范围

目标客户：ERP 使用少或 ERP 能力弱的中小型工厂。第一版解决“计划下达到工位、现场数据回收、完工入库回传”的闭环。

## 数据来源

| 数据 | MVP 来源 | 说明 |
| --- | --- | --- |
| 物料 | MES 基础档案手工维护 | 只维护可生产物料和必要原料，不做 ERP 全量同步 |
| BOM | MES 基础档案手工维护 | 只做一层 BOM，用于工单物料需求快照 |
| 工艺路线 | MES 基础档案手工维护 | 只维护工序、工位、标准时间 |
| 工单 | 页面新建、Excel/CSV 导入、轻量 ERP 接口 | ERP 只推工单，不同步完整主数据 |
| 报工 | 操作员页面录入 | 记录合格、不良、工时和异常补录 |
| 质检 | 质检页面录入 | 首检、巡检、终检和巡检停线 |
| 完工入库 | MES 入库关单 | ERP 来源工单生成本地回传记录 |

## 已落地能力

- 请求追踪：`request_id` / `trace_id`、结构化日志、CORS 暴露追踪头。
- 健康检查：`/health` 和带数据库探测的 `/health/ready`。
- 认证安全：Cookie token、退出失效、生产环境强制替换默认密钥。
- 工单主流程：创建、确认、派工、开工、报工、完成、入库关单、取消。
- 状态保护：取消在制品需要显式确认，工单状态由工序状态推导。
- 幂等保护：关键写入接口需要 `Idempotency-Key`，前端对 5xx/网络失败保留幂等键。
- Excel/CSV 导入：计划员可从 Excel 复制工单表导入，逐行返回成功/失败。
- ERP 轻量接入：ERP 推工单，MES 完工入库后保留本地回传记录并支持 ack。
- 追溯：工单维度串起创建、派工、报工、质检、入库和审计日志。
- 部署运维：生产 Compose、容器镜像、smoke check、备份、回滚和试点手册。

## 一期不做

- 不做物料、BOM、库存、工艺路线、质检明细的 ERP 全量同步。
- 不做复杂 APS 排产、设备采集、SPC、WMS 库存锁定。
- 不做多层 BOM 自动递归、替代料、返修路线、并行工序。
- 不做多工厂复杂组织架构，先按一个试点租户落地。

## 试点完成标准

1. 计划员能通过页面或 Excel/CSV 建 20 张以上真实工单。
2. 操作员能在手机或工位电脑完成开工、报工、暂停/恢复。
3. 质检能触发巡检停线，恢复后状态正确。
4. 至少 10 张工单完成入库并能查到完整追溯。
5. ERP 来源工单完工后，本地回传记录可查；ERP 恢复后能 ack。
6. 服务器重启后系统自动恢复，`/api/v1/health/ready` 返回 ready。
7. 完成一次数据库备份和一次演练回滚。
